# 应急最小实现（无表变更）

本页描述如何在 Anway 中使用“右键菜单 + 聊天工具”完成灾情文字上报、道路问题上报与态势总览。

## 使用方法

- 右键地图任意位置，选择：
  - 从此处出发（步行/驾车）：发送一条消息（后续将调用 AMap 工具，当前为占位）。
  - 上报灾情（文本）：输入文字后提交，系统会在该点位生成一个“未挂载”的点图层（例如 "Disaster report 2025-01-01 12:00:00"）。
  - 上报道路问题（文本）：与上同理，图层命名如 "Road issue ..."。
- “态势总览”按钮：点击后向聊天发送指令，系统汇总当前地图已挂载图层的数量、点线面分布，以及与“灾害/上报/设施”相关的关键图层概览。

## 显示上报结果

上报类图层默认不自动显示在地图上（未挂载）。
在聊天结果中会包含指示（kue_instructions），可直接调用 `add_layer_to_map`（由 AI 触发）将对应 layer_id 挂载并重命名为更易懂的名称。

## 设计原则

- 不新增数据库表与迁移；所有结果以矢量图层形式保存并复用现有渲染与样式链路。
- 仅通过 Pydantic 工具与聊天调用串接；保留现有 OSM 工具、上传流程与 WebSocket 体验。
- 时间维度先仅支持当前时刻（now），不引入历史时间线。

## 相关工具（由聊天自动调用）

- report_disaster_text(raw_text, lon, lat, current_time_iso)
- report_road_issue_text(raw_text, lon, lat, current_time_iso)
- summarize_situation(current_time_iso)

> 提示：如果未加载设施/灾害相关图层，可先用“连接远程/OSM 下载”等现有功能引入数据，再执行态势总览或后续路径规划。
